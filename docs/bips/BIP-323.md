## Abstract

A JSON-based protocol for describing automated trading strategies on BTC 5-minute prediction markets. A strategy is a declarative specification of five sequential decisions: **filter**, **signal**, **entry**, **risk**, and **exit**.

## Motivation

The biubiu.tools ecosystem requires a portable, human-readable format for strategy definitions that can be created in the UI, shared across endpoints, published on marketplaces, and executed by any conforming engine. Existing ad-hoc configuration formats are tightly coupled to specific implementations and cannot be transferred between systems.

BIP-323 provides a stable, versioned schema that:

1. Covers 100% of production strategies (27 verified) and 15 categories of real-world trading strategies.
2. Follows the trader's mental model: filter → signal → entry → risk → exit.
3. Is forward-compatible: new indicator types, filter types, and exit types can be added without breaking existing strategies.

## Specification

The key words "MUST", "MUST NOT", "REQUIRED", "SHALL", "SHOULD", "MAY" in this document are to be interpreted as described in RFC 2119.

### 1. Document Structure

A conforming strategy document is a JSON object with the following top-level fields:

```json
{
  "$schema": "BIP-323",
  "id": "<string>",
  "name": "<string | {zh, en}>",
  "signal": { "method": "<string>", "params": {} },
  "entry": { "amount": "<number>", "method": "<string>" }
}
```

All fields not listed as REQUIRED are OPTIONAL. Implementations MUST ignore unknown fields.

### 2. Meta Fields

| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| `$schema` | string | REQUIRED | MUST be `"BIP-323"` |
| `id` | string | REQUIRED | 3-40 characters, `[a-z0-9-]` only |
| `name` | string \| {zh, en} | REQUIRED | Display name. String for single language, object for bilingual |
| `description` | string \| {zh, en} | OPTIONAL | One-line summary |
| `tags` | string[] | OPTIONAL | Classification tags |
| `riskLevel` | string | OPTIONAL | One of: `"conservative"`, `"balanced"`, `"aggressive"` |
| `author` | string | OPTIONAL | Creator identifier |
| `version` | integer | OPTIONAL | Strategy iteration number, monotonically increasing |
| `createdAt` | string | OPTIONAL | ISO 8601 datetime |

### 3. Filter

OPTIONAL. Pre-conditions evaluated before signal computation. All configured filters MUST pass for the round to proceed. Filters do not determine direction.

```json
{
  "filter": {
    "schedule": { ... },
    "volatility": { ... },
    "trend": { ... }
  }
}
```

#### 3.1 schedule

| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| `timezone` | string | REQUIRED | IANA timezone identifier |
| `windows` | array | REQUIRED | Non-empty array of time windows |
| `windows[].days` | string[] | REQUIRED | Subset of: `"mon"`, `"tue"`, `"wed"`, `"thu"`, `"fri"`, `"sat"`, `"sun"` |
| `windows[].from` | string | REQUIRED | `"HH:MM"` format, `from` < `to` |
| `windows[].to` | string | REQUIRED | `"HH:MM"` format |

A round passes if current time falls within ANY window.

#### 3.2 volatility

| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| `mode` | string | REQUIRED | `"require_high"` or `"require_low"` |
| `atrThreshold` | number | REQUIRED | 0 < atrThreshold < 1. ATR as fraction of price |

When `mode` is `"require_high"`, round passes if ATR% >= threshold. When `"require_low"`, passes if ATR% <= threshold.

#### 3.3 trend

| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| `minEfficiencyRatio` | number | OPTIONAL | 0-1. Kaufman Efficiency Ratio threshold |
| `minBodyRatio` | number | OPTIONAL | 0-1. Average candle body/range ratio |
| `lookback` | integer | OPTIONAL | > 0. Default: 10. Candle lookback count |
| `mustMatchDirection` | boolean | OPTIONAL | Default: true. Trend direction must agree with signal |

### 4. Signal

REQUIRED. Determines trade direction (UP or DOWN) and whether to trade at all.

| Field | Type | Required |
|-------|------|----------|
| `method` | string | REQUIRED |
| `params` | object | Depends on method |

Exactly one method MUST be specified.

#### 4.1 clob_follow

Follow CLOB order book majority direction. No params.

#### 4.2 clob_fade

Opposite of CLOB majority direction. No params.

#### 4.3 fixed

Always trade in a fixed direction.

| Param | Type | Required | Constraints |
|-------|------|----------|-------------|
| `direction` | string | REQUIRED | `"up"` or `"down"` |

#### 4.4 candle

K-line based direction. Three modes.

| Param | Type | Required |
|-------|------|----------|
| `mode` | string | REQUIRED. One of: `"follow"`, `"streak"`, `"template"` |

**mode: "follow"**

| Param | Type | Default |
|-------|------|---------|
| `reverse` | boolean | false. When true, bet opposite of last candle |

**mode: "streak"**

| Param | Type | Required | Constraints |
|-------|------|----------|-------------|
| `count` | integer | REQUIRED | 2-10. Consecutive same-direction candles to trigger |
| `action` | string | REQUIRED | `"follow"` (trade with streak) or `"reverse"` (trade against) |

**mode: "template"**

| Param | Type | Required | Constraints |
|-------|------|----------|-------------|
| `sequence` | CandleShape[] | REQUIRED | 1-128 candle shapes |
| `direction` | string | REQUIRED | `"up"`, `"down"`, or `"reverse"` |
| `minSimilarity` | number | OPTIONAL | 0-1. Default: 0.75 |

CandleShape:

| Field | Type | Values |
|-------|------|--------|
| `direction` | string | `"bull"`, `"bear"`, `"any"` |
| `bodySize` | string | `"large"` (>70%), `"medium"` (30-70%), `"small"` (<30%), `"any"` |
| `upperWick` | string | `"long"` (>50% of range), `"short"`, `"any"` |
| `lowerWick` | string | `"long"`, `"short"`, `"any"` |

Similarity is computed as: `matched_non_any_features / total_non_any_features`. Features set to `"any"` are excluded from both numerator and denominator. If all features are `"any"`, the candle always matches.

When `direction` is `"reverse"`: use opposite direction of the last candle in the matched real sequence.

#### 4.5 rsi_reversal

| Param | Type | Required | Constraints |
|-------|------|----------|-------------|
| `oversold` | number | REQUIRED | 0-100. RSI <= this → UP |
| `overbought` | number | REQUIRED | 0-100. RSI >= this → DOWN |

Constraint: `oversold` < `overbought`. RSI between thresholds → skip round.

#### 4.6 scoring

Multi-indicator weighted scoring system.

| Param | Type | Required |
|-------|------|----------|
| `threshold` | number | REQUIRED. Total score >= this → trade |
| `allowReverse` | boolean | OPTIONAL. Default: false |
| `reverseThreshold` | number | OPTIONAL. Total score <= this → trade in reverse direction |
| `indicators` | Indicator[] | REQUIRED. 1-20 indicators |

Indicator:

| Field | Type | Required |
|-------|------|----------|
| `name` | string | REQUIRED |
| `type` | string | REQUIRED. See Indicator Types |
| `config` | object | OPTIONAL. Type-specific parameters |
| `rules` | Rule[] | REQUIRED. 1+ scoring rules |
| `required` | boolean | OPTIONAL. If true, at least one rule must match or round is skipped |

Rule:

| Field | Type | Description |
|-------|------|-------------|
| `when` | [number\|null, number\|null] | Value range. null = unbounded. Left-inclusive, right-exclusive |
| `score` | number | Direction-agnostic score |
| `up` | number | Score added when evaluating UP direction |
| `down` | number | Score added when evaluating DOWN direction |
| `veto` | boolean | If true and range matches, skip round immediately |

`score` and `up`/`down` are mutually exclusive. If both present, `up`/`down` take precedence.

**Indicator Types:**

Oscillators:

| Type | Config | Output |
|------|--------|--------|
| `rsi` | `{period: 14}` | 0-100 |
| `stochastic` | `{kPeriod: 14, dPeriod: 3}` | 0-100 |
| `macd` | `{fast: 12, slow: 26, signal: 9}` | real number |

Trend:

| Type | Config | Output |
|------|--------|--------|
| `roc` | `{period: 3}` | percentage |
| `ema_crossover` | `{fast: 9, slow: 21}` | 0 or 1 |
| `price_vs_ema` | `{period: 21}` | percentage |
| `adx` | `{period: 14}` | 0-100 |
| `efficiency_ratio` | `{period: 15}` | 0-1 |

Volatility:

| Type | Config | Output |
|------|--------|--------|
| `atr` | `{period: 14}` | real number |
| `atr_pct` | `{period: 14}` | percentage |
| `bollinger_pct` | `{period: 20, stddev: 2}` | 0-1 |
| `bollinger_width` | `{period: 20, stddev: 2}` | percentage |
| `candle_range_pct` | none | percentage |

Volume:

| Type | Config | Output |
|------|--------|--------|
| `volume_ratio` | `{period: 10}` | 0+ |
| `volume_trend` | `{recent: 3, prior: 3}` | ratio |
| `taker_buy_ratio` | `{period: 5}` | 0-1 |
| `vwap_deviation` | none | percentage |

Price Action:

| Type | Config | Output |
|------|--------|--------|
| `body_ratio` | `{period: 3}` | 0-1 |
| `bullish_candles` | `{period: 5}` | 0-1 |
| `wick_ratio` | none | -1 to 1 |
| `engulfing` | none | -1, 0, or 1 |
| `consecutive` | `{period: 5}` | integer |

Order Flow (requires `clob.obi`, `clob.tradeFlow`, etc.):

| Type | Config | Output |
|------|--------|--------|
| `obi` | none | [-1, 1]. Order book imbalance |
| `trade_flow` | none | [0, 1]. Buy-side trade volume ratio |
| `large_trade_dir` | none | [-1, 1]. Large trade net direction |
| `micro_momentum` | none | real. 5-second price acceleration |
| `price_divergence` | none | cents. Polymarket vs Binance price gap |

Futures (requires `futures.*`):

| `long_short_ratio` | none | ratio. Retail long/short accounts |
| `futures_taker` | none | [0, 1]. Futures taker buy ratio |

Cross-Asset (requires `correlatedAssets.*`, all from Binance public WebSocket):

| Type | Config | Output |
|------|--------|--------|
| `eth_btc_change` | none | %. ETH/BTC pair change in 5 min |

Implementations MAY support additional indicator types. Unknown types SHOULD be skipped during evaluation without error.

#### 4.7 ml

| Param | Type | Required |
|-------|------|----------|
| `model` | string | REQUIRED. Model identifier |
| `type` | string | REQUIRED. `"random_forest"`, `"xgboost"`, or `"gbdt"` |
| `minConfidence` | number | OPTIONAL. 0.5-1.0. Default: 0.6 |
| `waitForDataSec` | number | OPTIONAL. Default: 30. Seconds to collect data before prediction |

Model sources: built-in (`{type}-{version}-{window}`), user-uploaded (`user:{userId}:{name}`), marketplace (`market:{strategyId}:{name}`).

#### 4.8 post_close

| Param | Type | Required |
|-------|------|----------|
| `confidencePrice` | number | REQUIRED. Midpoint >= this → inferred winner |
| `timeoutSec` | number | OPTIONAL. Default: 30 |

Constraint: MUST be paired with `entry.method = "scan"` and `entry.windows = []`.

### 5. Entry

REQUIRED. Execution parameters.

| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| `amount` | number | REQUIRED | 0.01-10000. USD per round |
| `method` | string | REQUIRED | `"market"`, `"limit"`, `"swing"`, or `"scan"` |
| `maxPrice` | number | OPTIONAL | 0.01-0.99. Maximum share price |
| `minPrice` | number | OPTIONAL | 0.01-0.99. Default: 0.01. Minimum share price |
| `windows` | Window[] | OPTIONAL | 1-3 entry time windows |
| `maxPerHour` | integer | OPTIONAL | Rate limit |
| `sizing` | Sizing | OPTIONAL | Dynamic position sizing |

Window:

| Field | Type | Constraints |
|-------|------|-------------|
| `from` | integer | Remaining seconds when window opens. `from` > `to` |
| `to` | integer | Remaining seconds when window closes. >= 0 |

Method-specific fields:

| Method | Additional Fields |
|--------|-------------------|
| `market` | none |
| `limit` | `limitPrice` (REQUIRED), `timeoutSec` (OPTIONAL) |
| `swing` | `targetPrice` (REQUIRED), `windowSec` (OPTIONAL) |
| `scan` | `scanMaxPrice`, `scanBudget`, `scanDelaySec`, `scanDurationSec`, `scanIntervalMs` (all REQUIRED) |

Sizing:

| Field | Type | Constraints |
|-------|------|-------------|
| `method` | string | `"fixed"`, `"martingale"`, `"anti_martingale"`, or `"kelly"` |
| `multiplier` | number | Required for martingale/anti_martingale |
| `maxAmount` | number | Maximum amount after adjustment |
| `minAmount` | number | Minimum amount (kelly) |
| `resetOnWin` | boolean | Martingale: reset to base on win |
| `resetOnLoss` | boolean | Anti-martingale: reset to base on loss |
| `lookback` | integer | Kelly: rounds for win rate calculation |
| `fraction` | number | Kelly: fraction of optimal (0.5 = half-Kelly) |

When `sizing` is absent, amount is fixed.

### 6. Risk

OPTIONAL. Protection mechanisms.

| Field | Type | Description |
|-------|------|-------------|
| `dailyLossLimit` | number | Max USD loss per day. Strategy pauses when reached |
| `streakRules` | StreakRule[] | Behavioral rules on win/loss streaks |
| `hedge` | Hedge | Opposite-direction insurance |
| `stopLoss` | StopLoss | Early exit on price crash |

StreakRule:

| Field | Type | Required | Constraints |
|-------|------|----------|-------------|
| `trigger` | string | REQUIRED | `"win"` or `"loss"` |
| `count` | integer | REQUIRED | >= 2 |
| `action` | string | REQUIRED | `"pause"`, `"stop"`, or `"adjust_amount"` |
| `pauseMinutes` | integer | When action=pause | 1-1440 |
| `multiply` | number | When action=adjust_amount | Multiplier for amount |
| `maxAmount` | number | When action=adjust_amount | Upper bound |

Hedge:

| Field | Type | Required |
|-------|------|----------|
| `enabled` | boolean | REQUIRED |
| `price` | number | REQUIRED. 0.001-0.99 |
| `shares` | integer | REQUIRED. 1-10000 |
| `sellAbove` | number | REQUIRED. Sell threshold |

StopLoss:

| Field | Type | Required |
|-------|------|----------|
| `price` | number | REQUIRED. Trigger price |
| `minHoldSec` | integer | OPTIONAL. Minimum hold before SL can trigger |

### 7. Exit

OPTIONAL. Array of exit rules evaluated in order. First triggered wins. If absent or empty, `[{"type": "settlement"}]` is implied.

| Type | Fields | Trigger |
|------|--------|---------|
| `takeProfit` | `profitPct` (REQUIRED, 0-1) | Position profit >= profitPct |
| `trailingStop` | `drawdownPct` (REQUIRED, 0-1) | Drawdown from peak >= drawdownPct |
| `checkpoint` | `atRemainingSec` (REQUIRED, > 0) | Re-evaluate signal at N seconds remaining; exit if reversed |
| `settlement` | none | Wait for round settlement |

Implementations MUST append `settlement` if not present in the array.

### 8. Validation Rules

A conforming implementation MUST reject a strategy document that violates any of the following:

1. `$schema` MUST be `"BIP-323"`.
2. `id` MUST match `^[a-z0-9-]{3,40}$`.
3. `signal.method` MUST be a recognized method or the document MUST be rejected.
4. `signal.method = "post_close"` REQUIRES `entry.method = "scan"` AND `entry.windows = []`.
5. `signal.method = "rsi_reversal"` REQUIRES `params.oversold < params.overbought`.
6. `signal.method = "ml"` REQUIRES `params.model` AND `params.type`.
7. `signal.method = "candle"` with `mode = "template"` REQUIRES `params.sequence` with 1-128 items.
8. `signal.method = "fixed"` REQUIRES `params.direction`.
9. `entry.method = "limit"` REQUIRES `entry.limitPrice`.
10. `entry.method = "swing"` REQUIRES `entry.targetPrice`.
11. `entry.method = "scan"` REQUIRES `scanMaxPrice`, `scanBudget`, `scanDelaySec`, `scanDurationSec`, `scanIntervalMs`.
12. `entry.windows[].from` MUST be > `entry.windows[].to`.
13. `entry.minPrice` MUST be < `entry.maxPrice` when both are present.
14. `risk.stopLoss.price` SHOULD be < `entry.maxPrice`.
15. `filter.schedule.windows[].from` MUST be < `windows[].to` (chronological order).
16. Unknown fields MUST be ignored by the implementation.

### 9. Minimal Valid Document

```json
{
  "$schema": "BIP-323",
  "id": "simple-follow",
  "name": "Simple Follow",
  "signal": { "method": "clob_follow" },
  "entry": { "amount": 5, "method": "market" }
}
```

### 10. Extensibility

New indicator types, filter types, and exit types MAY be added in future BIPs without modifying this specification. Implementations encountering unknown indicator types within a `scoring` signal SHOULD skip the unknown indicator without error. Implementations encountering an unknown `signal.method` or `entry.method` MUST reject the document.

New top-level fields MAY be added by future BIPs. Implementations MUST ignore unknown top-level fields per Rule 16.

## Rationale

**Why five sections?** The filter→signal→entry→risk→exit pipeline mirrors the sequential decisions a trader makes each round. Alternative structures (e.g., grouping hedge with entry, or exit with risk) were considered but rejected because they conflate distinct concerns.

**Why `scoring` over multiple signal methods?** Allowing only one signal method per strategy avoids ambiguity about how to combine conflicting signals. The `scoring` method provides a principled composition mechanism via weighted indicators.

**Why `streakRules` over `cooldown`?** A simple cooldown only handles loss streaks. Real traders also manage win streaks (reducing overconfidence) and adjust position sizes based on recent performance. `streakRules` generalizes this with a trigger/action pattern.

**Why `candle.template` over named patterns?** Named patterns (e.g., "hammer", "engulfing") are subjective — different traders define them differently. The template approach decomposes patterns into objective features (direction, body size, wick length) that can be precisely specified and matched.

**Why `minSimilarity` excludes `"any"` features?** If `"any"` features counted toward the denominator, a template with many wildcards would have artificially inflated similarity scores, making the threshold meaningless.

## Backwards Compatibility

BIP-323 is a new specification. Existing v2 strategy configurations (`polymarket-strategy-v2`) can be mechanically converted:

| v2 Field | BIP-323 Field |
|----------|---------------|
| `$schema: "polymarket-strategy-v2"` | `$schema: "BIP-323"` |
| `direction` | `signal` |
| `direction.method: "signal_score"` | `signal.method: "scoring"` |
| `direction.method: "ml_model"` | `signal.method: "ml"` |
| `direction.method: "post_market_inference"` | `signal.method: "post_close"` |
| `direction.method: "prev_candle"` | `signal.method: "candle"`, `params.mode: "follow"` |
| `direction.params.thresholdLow/High` | `signal.params.oversold/overbought` |
| `gates[].type: "daily_loss_limit"` | `risk.dailyLossLimit` |
| `gates[].type: "cooldown"` | `risk.streakRules[]` |
| `gates[].type: "volatility"` | `filter.volatility` |
| `gates[].type: "trend"` | `filter.trend` |
| `gates[].type: "schedule"` | `filter.schedule` |
| `hedge` (top-level) | `risk.hedge` |
| `exit[].type: "stop_loss"` | `risk.stopLoss` |
| `exit[].type: "take_profit"` | `exit[].type: "takeProfit"` |
| `exit[].type: "trailing_stop"` | `exit[].type: "trailingStop"` |
| `entry.method: "swing_limit"` | `entry.method: "swing"` |
| `entry.method: "post_market_scan"` | `entry.method: "scan"` |
| `label: {zh, en}` | `name: string \| {zh, en}` |
| `signals.rules[].conditions[].scoreUp/Down` | `rules[].up/down` |

## Security Considerations

Strategy documents SHOULD be validated against the rules in Section 8 before execution. Implementations MUST NOT execute arbitrary code embedded in strategy documents. The `ml.model` field references a model by identifier; implementations MUST NOT interpret it as a file path or URL for direct execution.
