# SvelteKit Next-Gen i18n 库设计文档 (PRD)

## 1. 项目概述

为 SvelteKit 5 开发者提供一个**极致轻量、金融级精准、开发体验极佳**的国际化解决方案。突破传统 i18n 库仅做"文字翻译"的局限，深度适配复杂业务（如 DeFi/国际贸易）中的混合偏好习惯。

---

## 2. 核心特性

- **路由级按需加载**：消息文件与路由 1:1 映射，自动按需加载
- **5 维度偏好模型**：语言与格式习惯完全解耦
- **金融级精度**：原生 BigInt 支持，零精度损失
- **Alt+Click 溯源**：点击文本直接跳转到 JSON 源文件对应行
- **类型安全**：自动生成类型定义，完整代码补全
- **Svelte 5 Runes**：使用 `$state`、`$derived` 实现响应式

---

## 3. 消息文件格式

### 3.1 扁平 JSON 结构

采用扁平 key-value 格式，一行一个翻译项，拒绝嵌套：

```jsonc
// messages/zh/common.json
{
  "nav.home": "首页",
  "nav.dashboard": "控制台",
  "nav.settings": "设置"
}

// messages/zh/dashboard.json
{
  "title": "控制台",
  "welcome": "欢迎回来，{name}",
  "stats.users": "用户数",
  "stats.revenue": "收入"
}

// messages/zh/user/[id].json  ← 支持动态路由
{
  "profile.title": "用户资料",
  "profile.followers": "{count, plural, =0 {暂无粉丝} =1 {1 位粉丝} other {# 位粉丝}}"
}
```

### 3.2 目录结构

```
messages/
├── zh/
│   ├── common.json              # 全局（所有页面）
│   ├── dashboard.json           # /dashboard
│   ├── dashboard/_layout.json   # /dashboard/* 共享
│   ├── dashboard/settings.json  # /dashboard/settings
│   └── user/[id].json           # /user/:id
├── en/
│   └── ...
```

### 3.3 加载合并策略

访问 `/dashboard/settings` 时自动加载并合并：

```
common.json → dashboard/_layout.json → dashboard/settings.json
```

查找优先级：`Page Specific > Layout Specific > Global Common`

## 4. 5 维度偏好模型

解耦"语言"与"习惯"，允许用户自由组合：

| 维度                   | 作用                       | 示例                        |
| :--------------------- | :------------------------- | :-------------------------- |
| **locale**       | UI 语言，决定加载哪个 JSON | `zh-CN`                   |
| **numberLocale** | 千分位、小数点符号习惯     | `de-DE` → `1.234,56`   |
| **currency**     | 货币符号                   | `EUR` → `€`           |
| **dateLocale**   | 日期格式习惯               | `en-US` → `MM/DD/YYYY` |
| **timezone**     | 时区（含夏令时自动处理）   | `Asia/Shanghai`           |

典型场景：中国用户看中文界面，但使用欧元结算、德国数字格式。

## 5. API 设计

### 5.1 初始化

```typescript
// src/lib/i18n.ts
import { createI18n } from '@shelchin/i18n-sveltekit';

export const i18n = createI18n({
  defaultLocale: 'zh-CN',
  fallbackLocale: 'en',
  preferences: {
    numberLocale: 'zh-CN',
    currency: 'CNY',
    dateLocale: 'zh-CN',
    timezone: 'Asia/Shanghai',
  }
});
```

### 5.2 SvelteKit 集成

```typescript
// src/hooks.server.ts
import { i18n } from '$lib/i18n';

export const handle = i18n.handle({
  detectLocale: ({ url, cookies, request }) => {
    // 优先级：URL > Cookie > Accept-Language > Fallback
    return url.pathname.split('/')[1]
      || cookies.get('locale')
      || request.headers.get('accept-language')?.split(',')[0]
      || 'zh-CN';
  }
});
```

```typescript
// src/routes/+layout.ts
import { i18n } from '$lib/i18n';

export const load = i18n.load;  // 自动按路由加载消息
```

### 5.3 组件中使用（Svelte 5 Runes）

```svelte
<script>
  import { t, formatNumber, formatDate, locale, preferences } from '$lib/i18n';

  let amount = 1234567890123456789n;  // BigInt
  let now = new Date();
</script>

<!-- 基础翻译 -->
<h1>{t('dashboard.title')}</h1>

<!-- 带插值 -->
<p>{t('welcome', { name: 'Tom' })}</p>

<!-- 复数处理 -->
<p>{t('profile.followers', { count: 3 })}</p>

<!-- BigInt 金额格式化 -->
<p>{formatNumber(amount, { style: 'currency' })}</p>
<!-- 输出: ¥1,234,567,890,123,456,789 -->

<!-- 日期格式化（跟随时区偏好） -->
<p>{formatDate(now, { dateStyle: 'full', timeStyle: 'short' })}</p>
<!-- 输出: 2026年2月9日星期日 14:30 -->

<!-- 切换语言 -->
<button onclick={() => locale.set('en')}>English</button>
```

### 5.5 数字输入解析

```svelte
<script>
  import { parseNumber } from '$lib/i18n';

  let displayValue = $state('');  // 用户看到的：1.234,56
  let realValue = $derived(parseNumber(displayValue));  // 实际值：1234.56
</script>

<input bind:value={displayValue} />
<p>实际值: {realValue}</p>
```

## 6. 金融级数字引擎

### 6.1 BigInt 格式化

```typescript
formatNumber(1234567890123456789n, { style: 'currency' })
// ¥1,234,567,890,123,456,789

formatNumber('0.00000000456', { significantDigits: 3 })
// 0.00000000456（保留有效数字）
```

### 6.2 逆向解析

根据用户的 `numberLocale` 偏好，将本地化格式还原为标准数字：

```typescript
// 用户使用德语格式
parseNumber('1.234,56')  // → '1234.56'

// 用户使用法语格式
parseNumber('1 234,56')  // → '1234.56'
```

## 8. 动态路由支持

### 8.1 自动生成路由映射

```typescript
// 自动生成的 $i18n/routes.ts
export const routeMessages = {
  '/dashboard': ['common', 'dashboard'],
  '/dashboard/settings': ['common', 'dashboard/_layout', 'dashboard/settings'],
  '/user/[id]': ['common', 'user/[id]'],
  '/user/[id]/profile': ['common', 'user/[id]', 'user/[id]/profile'],
} as const;
```

### 8.2 运行时匹配

```typescript
function matchRoute(pathname: string): string[] {
  // 1. 精确匹配
  if (routeMessages[pathname]) return routeMessages[pathname];

  // 2. 动态路由匹配
  for (const pattern of Object.keys(routeMessages)) {
    const regex = patternToRegex(pattern);
    if (regex.test(pathname)) return routeMessages[pattern];
  }

  return ['common'];
}

function patternToRegex(pattern: string): RegExp {
  return new RegExp(
    '^' + pattern
      .replace(/\[\.\.\.(\w+)\]/g, '(.+)')   // [...slug]
      .replace(/\[(\w+)\]/g, '([^/]+)')      // [id]
    + '$'
  );
}
```

## 9. 技术实现

### 9.1 模块结构

```
packages/i18n/
├── core/
│   ├── state.svelte.ts       # Svelte 5 runes 状态管理
│   ├── translator.ts         # t() 函数
│   └── loader.ts             # 消息加载器
├── format/
│   ├── number.ts             # BigInt 数字引擎
│   ├── date.ts               # 日期格式化
│   └── parse.ts              # 逆向解析
├── devtools/
│   ├── source-map.ts         # 行号追踪
│   └── click-handler.ts      # Alt+Click 处理
├── vite-plugin/
│   ├── index.ts              # 插件入口
│   ├── scanner.ts            # 扫描 messages/
│   ├── line-indexer.ts       # JSON 行号索引
│   └── type-gen.ts           # 类型生成
└── sveltekit/
    ├── hook.ts               # handle hook
    └── load.ts               # load 辅助函数
```

### 9.2 技术栈

- **格式化引擎**：浏览器原生 `Intl` API
- **构建工具**：Vite 插件
- **状态管理**：Svelte 5 Runes (`$state`, `$derived`)
- **JSON 解析**：`jsonc-parser`（用于行号索引）
